## File-by-File Changes

### FILE: docker-compose.yml
**Remove:**
- `ports:` section entirely (no port mapping in macvlan mode)
**Add:**
- `networks: [macvlan_net]`
- `ipam` config under service (or rely on network-level IPAM + static IP assignment)

Actually, docker compose v2 with macvlan external network needs a bit more care. The static IP is assigned at the service level or network level.

**Final docker-compose.yml structure:**
```yaml
services:
  openspeedtest:
    image: openspeedtest/latest
    container_name: openspeedtest
    restart: unless-stopped
    environment:
      - MODE=standalone
    networks:
      macvlan_net:
        ipv4_address: 192.168.1.20

networks:
  macvlan_net:
    external: true
```

**NOTE:** The external macvlan network must already exist on the host with matching IPAM config (subnet/gateway). The `ipv4_address` in the service is optional if the external network already has the pool — but explicit is safer.

## Risks / Gotchas

1. **Parent interface mismatch:** The Docker macvlan network must use parent=macvlan0, NOT eth0. If the existing network has parent=eth0, the play must recreate it.
2. **IPAM conflict:** If the external macvlan network already exists with a different subnet/gateway, the compose static IP may conflict. The play should idempotently handle this.
3. **OpenSpeedTest port:** The container exposes port 3000 (web UI), not 80. Verify with `docker inspect` if uncertain.
4. **LXC network config:** Must confirm LXC hlh-docker has its net0 interface on macvlan0 (not vmbr0). This was set in the previous session — verify before deploy.
5. **Firewall:** No firewall should block 192.168.1.20:3000 — macvlan bypasses the host's iptables.
